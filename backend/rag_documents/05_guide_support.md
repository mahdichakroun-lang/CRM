# 🎫 Guide du Support Client — FRS IT Development Company

## Comment obtenir de l'aide

### 1. Portail Client (recommandé)
Le moyen le plus rapide pour obtenir du support est d'utiliser votre **portail client** :
1. Connectez-vous à votre espace client
2. Allez dans **"Mes Tickets"**
3. Cliquez sur **"Nouveau Ticket"**
4. Remplissez les détails et soumettez

### 2. Par email
Envoyez votre demande à **support@frs.tn** en précisant :
- Votre nom et entreprise
- Le projet concerné
- Une description détaillée du problème
- Si possible, des captures d'écran

### 3. Par téléphone
Appelez le **+216 71 000 000** pendant nos horaires de support :
- **Dimanche — Jeudi** : 8h00 — 17h00
- **Samedi** : pour les urgences uniquement (formule Premium)

## Catégories de tickets

Lorsque vous créez un ticket, choisissez la catégorie appropriée :

| Catégorie | Quand l'utiliser | Exemples |
|-----------|-----------------|----------|
| **🐛 Bug** | Quelque chose ne fonctionne pas correctement | Erreur 500, page qui plante, calcul incorrect |
| **🛟 Support** | Besoin d'aide ou d'assistance | Comment faire X ? Aide à la configuration |
| **❓ Question** | Question générale sur votre projet | Délais, fonctionnalités prévues |
| **✨ Demande de fonctionnalité** | Vous souhaitez une nouvelle fonctionnalité | Ajouter un export Excel, nouveau rapport |
| **⚠️ Incident** | Problème grave affectant la production | Application inaccessible, perte de données |
| **📌 Autre** | Tout ce qui ne rentre pas dans les autres catégories | Demande administrative, question sur la facturation |

---

## Niveaux de priorité

| Priorité | Impact | Exemple |
|----------|--------|---------|
| **🔴 Urgente** | Application totalement inaccessible | Serveur en panne, impossible de se connecter |
| **🟠 Haute** | Fonctionnalité majeure non opérationnelle | Module de facturation en panne, exports bloqués |
| **🔵 Moyenne** | Problème gênant mais avec contournement possible | Affichage incorrect, lenteur ponctuelle |
| **🟢 Basse** | Demande d'amélioration ou question | Changer un libellé, ajouter un champ |

## Processus de traitement

```
Ticket créé (OUVERT)
    ↓
Prise en charge par un agent (EN COURS)
    ↓
Besoin d'info → EN ATTENTE CLIENT
    ↓
Résolution trouvée (RÉSOLU)
    ↓
Validation client → FERMÉ
```

### Statuts des tickets

| Statut | Signification |
|--------|---------------|
| **Ouvert** | Votre ticket a été reçu et sera traité prochainement |
| **En cours** | Un agent travaille activement sur votre demande |
| **En attente client** | Nous avons besoin d'une information de votre part |
| **Résolu** | La solution a été appliquée |
| **Fermé** | Le ticket est finalisé et clôturé |

## Bonnes pratiques pour un ticket efficace

Pour accélérer la résolution de votre problème, veuillez :

1. **Titre clair et précis** : décrivez le problème en une phrase
   - ✅ *« Erreur lors de la génération du rapport de ventes »*
   - ❌ *« Ça ne marche pas »*

2. **Description détaillée** :
   - Qu'essayez-vous de faire ?
   - Quel est le résultat attendu ?
   - Quel est le résultat obtenu ?
   - Depuis quand le problème se produit ?

3. **Captures d'écran** : ajoutez des images quand c'est possible

4. **Étapes pour reproduire** : décrivez la séquence d'actions

5. **Bonne catégorie et priorité** : choisissez-les avec soin pour un traitement optimal
