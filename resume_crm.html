CRM Internal — Résumé des Fonctionnalités

Application de Gestion de la Relation Client | Version 1.0.0
Stack : FastAPI · React · MySQL 8 · Docker | Avril 2026

🏗️ 1. Présentation Générale

CRM Internal est une plateforme unifiée de gestion commerciale et de support client, développée pour une SSII (FRS). Elle couvre le cycle de vente complet (Leads → Deals → Devis) et un helpdesk intégré (Tickets), avec un portail client dédié et un chatbot IA.

Architecture : Frontend React (Vite + TailwindCSS) → API REST FastAPI → MySQL 8, le tout orchestré par Docker Compose (3 conteneurs : frontend, backend, db).

🔑 2. Authentification & Rôles (RBAC)

Authentification par JWT (JSON Web Token) avec expiration de 60 minutes. 5 rôles avec permissions différentes :

RôleAccèsRestrictions
AdminAccès total à toutes les fonctionnalitésAucune
ManagerCRM complet + gestion utilisateurs—
CommercialLeads, Accounts, Contacts, Deals, Quotes, ActivitiesPas d'accès à la gestion des utilisateurs
SupportTickets, Activities + lecture entités commercialesPas de modification des données commerciales
ClientPortail Client uniquement (ses tickets, ses devis)Accès limité à son propre compte

🤖 14. Chatbot IA (Assistant CRM)

Technologie : RAG (Retrieval-Augmented Generation) avec 58 chunks de documents indexés + Groq LLM pour la génération de réponses intelligentes en français.
  • Widget flottant en bas à droite du portail client
  • Message d'accueil automatique
  • Répond aux questions sur les services, devis, support
  • Bouton STOP pour arrêter la génération

🗺️ Parcours Utilisateur (Démarche du Site)

1. Connexion → 2. Dashboard (vue d'ensemble) → 3. Créer un Lead (prospect)
→ 4. Qualifier le Lead → 5. Convertir en Account + Contact + Deal
→ 6. Gérer le Deal dans le Pipeline (5 étapes) → 7. Créer un Devis
→ 8. Planifier des Activités (appels, réunions) → 9. Gagner le Deal ✅

Support : Client crée un Ticket → Support traite → Messages → Résolution → Email
Suivi : Tout est tracé dans le Journal d'Audit + Rapports temps réel
